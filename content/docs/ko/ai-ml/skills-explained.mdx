스킬을 소개한 이후로, Claude의 에이전트 생태계 다양한 구성 요소들이 어떻게 함께 작동하는지에 대한 관심이 있었습니다.

Claude Code에서 정교한 워크플로우를 구축하든, API로 엔터프라이즈 솔루션을 만들든, Claude.ai에서 생산성을 극대화하든, 어떤 도구를 언제 사용해야 하는지 아는 것은 Claude와의 작업 방식을 변화시킬 수 있습니다.

이 가이드는 각 구성 요소를 분해하고, 언제 무엇을 사용해야 하는지 설명하며, 강력한 에이전트 워크플로우를 위해 어떻게 결합할 수 있는지 보여줍니다.

## 에이전트 구성 요소 이해하기

### **스킬이란 무엇인가요?**

스킬은 지침, 스크립트, 리소스가 포함된 폴더로, 작업과 관련이 있을 때 Claude가 동적으로 발견하고 로드합니다. 스킬을 전문 교육 매뉴얼로 생각할 수 있습니다. 이는 Claude에게 특정 도메인의 전문 지식을 제공합니다 - Excel 스프레드시트 작업부터 조직의 브랜드 가이드라인 준수까지.

**스킬의 작동 방식:** Claude가 작업을 만나면 관련 일치 항목을 찾기 위해 사용 가능한 스킬을 스캔합니다. 스킬은 점진적 공개를 사용합니다: 메타데이터가 먼저 로드되어(~100 토큰), Claude가 스킬이 언제 관련 있는지 알 수 있도록 충분한 정보를 제공합니다. 전체 지침은 필요할 때 로드되고(&lt;5k 토큰), 번들된 파일이나 스크립트는 필요할 때만 로드됩니다.

**스킬을 사용해야 할 때:** Claude가 특수화된 작업을 일관되고 효율적으로 수행하기를 원할 때 스킬을 선택하세요. 스킬은 다음에 이상적입니다:

- **조직 워크플로우:** 브랜드 가이드라인, 규정 준수 절차, 문서 템플릿
- **도메인 전문 지식:** Excel 수식, PDF 조작, 데이터 분석
- **개인 선호도:** 필기 시스템, 코딩 패턴, 연구 방법

**예시:** 회사의 색상 팔레트, 타이포그래피 규칙, 레이아웃 사양을 포함하는 브랜드 가이드라인 스킬을 만드세요. Claude가 프레젠테이션이나 문서를 만들 때, 매번 설명할 필요 없이 자동으로 이러한 표준을 적용합니다.

스킬에 대한 자세한 내용을 알아보고 성장하는 스킬 라이브러리를 확인하세요.

### **프롬프트란 무엇인가요?**

프롬프트는 대화 중에 자연어로 Claude에게 제공하는 지침입니다. 이들은 일시적이고, 대화적이며, 반응적입니다 - 즉시 컨텍스트와 방향을 제공합니다.

**프롬프트를 사용해야 할 때:** 프롬프트를 다음에 사용하세요:

- 일회성 요청: "이 문서를 요약해주세요"
- 대화적 수정: "어조를 더 전문적으로 만들어주세요"
- 즉각적 컨텍스트: "이 데이터를 분석하고 추세를 식별해주세요"
- 임시 지침: "글머리 기호 목록으로 형식화해주세요"

**예시:**

> 이 코드에 대한 포괄적인 보안 검토를 수행해주세요. 다음을 찾고 있습니다:
>
> 1. 다음을 포함한 일반적인 취약점:
>    - 인젝션 결함 (SQL, 명령, XSS 등)
>    - 인증 및 권한 부여 문제
>    - 민감한 데이터 노출
>    - 보안 구성 오류
>    - 손상된 접근 제어
>    - 암호화 실패
>    - 입력 유효성 검사 문제
>    - 오류 처리 및 로깅 문제
>
> 2. 발견한 각 문제에 대해 다음을 제공해주세요:
>    - 심각도 수준 (치명적/높음/중간/낮음)
>    - 코드 내 위치 (줄 번호 또는 함수 이름)
>    - 보안 위험이 되는 이유와 악용될 수 있는 방법에 대한 설명
>    - 가능한 경우 코드 예제와 함께 구체적인 수정 권장사항
>    - 유사한 문제를 방지하기 위한 모범 사례 지침
>
> 3. 코드 컨텍스트: [코드가 하는 일, 언어/프레임워크, 실행 환경 설명 - 예: "이것은 사용자 인증을 처리하고 결제 데이터를 처리하는 Node.js REST API입니다"]
>
> 4. 추가 고려사항:
>    - OWASP Top 10 취약점이 있나요?
>    - 코드가 [특정 프레임워크/언어]에 대한 보안 모범 사례를 따르나요?
>    - 알려진 취약점이 있는 종속성이 있나요?
>
> 심각도와 잠재적 영향에 따라 결과의 우선순위를 정해주세요.

**전문가 팁:** 프롬프트는 Claude와 상호작용하는 주요 방법이지만, 대화 간에 지속되지 않습니다. 반복적인 워크플로우나 전문 지식의 경우, 프롬프트를 스킬이나 프로젝트 지침으로 캡처하는 것을 고려하세요.

**대신 스킬을 사용해야 할 때:** 여러 대화에서 동일한 프롬프트를 반복적으로 입력하는 것을 발견한다면, 이제 스킬을 만들 때입니다. "OWASP 표준을 사용하여 이 코드의 보안 취약점을 검토하세요"나 "이 분석을 실행 요약, 주요 발견 사항, 권장사항으로 형식화하세요"와 같은 반복적인 지침을 스킬로 변환하세요. 이는 매번 절차를 다시 설명하는 것을 피하고 일관된 실행을 보장합니다.

프롬프트 라이브러리, 프롬프트 모범 사례, 또는 스마트 프롬프트 메이커를 확인하여 시작하세요.

### **프로젝트란 무엇인가요?**

모든 유료 Claude 플랜에서 사용 가능한 프로젝트는 자체 채팅 기록과 지식 기반을 가진 독립적인 작업 공간입니다. 각 프로젝트는 문서를 업로드하고, 컨텍스트를 제공하며, 해당 프로젝트 내 모든 대화에 적용되는 사용자정의 지침을 설정할 수 있는 200K 컨텍스트 창을 포함합니다.

**프로젝트의 작동 방식:** 프로젝트 지식 기반에 업로드하는 모든 것이 해당 프로젝트 내 모든 채팅에서 사용 가능해집니다. Claude는 자동으로 이 컨텍스트를 사용하여 더 정보 있고 관련성 있는 응답을 제공합니다. 프로젝트 지식이 컨텍스트 제한에 접근하면, Claude는 RAG(Retrieval Augmented Generation) 모드를 원활하게 활성화하여 용량을 최대 10배까지 확장합니다.

**프로젝트를 사용해야 할 때:** 다음이 필요할 때 프로젝트를 선택하세요:

- **지속적 컨텍스트:** 모든 대화에 영향을 미쳐야 하는 배경 지식
- **작업 공간 구성:** 다른 이니셔티브를 위한 별도의 컨텍스트
- **팀 협업:** 공유 지식 및 대화 기록 (Team 및 Enterprise 플랜에서)
- **사용자정의 지침:** 프로젝트별 어조, 관점, 또는 접근 방식

**예시:** 시장 조사, 경쟁사 분석, 제품 사양을 포함하는 "Q4 제품 출시" 프로젝트를 만드세요. 이 프로젝트의 모든 채팅은 매번 다시 업로드하거나 컨텍스트를 다시 설명할 필요 없이 이 지식에 접근할 수 있습니다.

**대신 스킬을 사용해야 할 때:** 여러 에이전트나 대화가 동일한 전문 지식이 필요할 때 - 보안 검토 절차나 데이터 분석 방법 같은 - 개별 서브에이전트에 그 지식을 구축하는 대신 스킬을 만드세요. 스킬은 이식 가능하고 재사용 가능하며, 서브에이전트는 특정 워크플로우용으로 맞춤 제작됩니다. 모든 에이전트가 적용할 수 있는 전문 지식을 가르치기 위해 스킬을 사용하세요; 특정 도구 권한과 컨텍스트 격리로 독립적인 작업 실행이 필요할 때 서브에이전트를 사용하세요.

프로젝트에 대한 자세한 내용을 알아보세요.

### **서브에이전트란 무엇인가요?**

서브에이전트는 자체 컨텍스트 창, 사용자정의 시스템 프롬프트, 특정 도구 권한을 가진 전문 AI 어시스턴트입니다. Claude Code와 Claude Agent SDK에서 사용 가능하며, 서브에이전트는 개별 작업을 독립적으로 처리하고 결과를 메인 에이전트에게 반환합니다.

**서브에이전트의 작동 방식:** 각 서브에이전트는 자체 구성으로 작동합니다 - 무엇을 하고, 문제를 어떻게 접근하며, 어떤 도구에 접근할 수 있는지 정의합니다. Claude는 설명을 기반으로 작업을 적절한 서브에이전트에게 자동으로 위임하거나, 특정 서브에이전트를 명시적으로 요청할 수 있습니다.

**서브에이전트를 사용해야 할 때:** 서브에이전트를 다음에 사용하세요:

- **작업 전문화:** 코드 검토, 테스트 생성, 보안 감사
- **컨텍스트 관리:** 전문화된 작업을 오프로드하면서 메인 대화에 집중 유지
- **병렬 처리:** 여러 서브에이전트가 동시에 다양한 측면에서 작업
- **도구 제한:** 특정 서브에이전트를 안전한 작업으로 제한 (예: 읽기 전용 액세스)

**예시:**

> Write나 Edit 도구 없이 Read, Grep, Glob 도구에 접근 권한이 있는 code-reviewer 서브에이전트를 만드세요. 코드를 수정할 때, Claude는 의도하지 않은 코드 변경의 위험 없이 품질 및 보안 검토를 위해 자동으로 이 서브에이전트에게 위임합니다.

**대신 스킬을 사용해야 할 때:** 여러 에이전트나 대화가 동일한 전문 지식이 필요할 때 - 보안 검토 절차나 데이터 분석 방법 같은 - 개별 서브에이전트에 그 지식을 구축하는 대신 스킬을 만드세요. 스킬은 이식 가능하고 재사용 가능하며, 서브에이전트는 특정 워크플로우용으로 맞춤 제작됩니다. 모든 에이전트가 적용할 수 있는 전문 지식을 가르치기 위해 스킬을 사용하세요; 특정 도구 권한과 컨텍스트 격리로 독립적인 작업 실행이 필요할 때 서브에이전트를 사용하세요.

서브에이전트에 대한 자세한 내용을 알아보세요.

### **MCP란 무엇인가요?**

MCP는 AI 애플리케이션과 기존 도구 및 데이터 소스 간에 범용 연결 계층을 만듭니다.

Model Context Protocol (MCP)은 데이터가 있는 외부 시스템 - 콘텐츠 저장소, 비즈니스 도구, 데이터베이스, 개발 환경 - 에 AI 어시스턴트를 연결하기 위한 개방형 표준입니다.

**MCP의 작동 방식:** MCP는 Claude를 도구와 데이터 소스에 연결하는 표준화된 방법을 제공합니다. 각 데이터 소스에 대한 사용자정의 통합을 구축하는 대신, 단일 프로토콜에 대해 구축합니다. MCP 서버는 데이터와 기능을 노출하고; MCP 클라이언트(Claude 등)는 이러한 서버에 연결합니다.

**MCP를 사용해야 할 때:** Claude가 다음을 수행하기를 원할 때 MCP를 선택하세요:

- 외부 데이터 액세스: Google Drive, Slack, GitHub, 데이터베이스
- 비즈니스 도구 사용: CRM 시스템, 프로젝트 관리 플랫폼
- 개발 환경 연결: 로컬 파일, IDE, 버전 제어
- 사용자정의 시스템 통합: 귀하의 독점 도구 및 데이터 소스

**예시:** MCP를 통해 Claude를 회사의 Google Drive에 연결하세요. 이제 Claude는 수동 업로드 없이 문서를 검색하고, 파일을 읽고, 내부 지식을 참조할 수 있습니다 - 연결은 지속되고 자동으로 업데이트됩니다.

**대신 스킬을 사용해야 할 때:** MCP는 Claude를 데이터에 연결합니다; 스킬은 Claude에게 그 데이터로 무엇을 할지 가르칩니다. 도구 사용법이나 절차를 설명하고 있다면 - 예를 들어 "데이터베이스를 쿼리할 때, 항상 먼저 날짜 범위로 필터링하세요"나 "이러한 특정 수식으로 Excel 보고서를 형식화하세요" - 그것은 스킬입니다. Claude가 데이터베이스나 Excel 파일에 액세스해야 한다면, 그것은 MCP입니다. 둘을 함께 사용하세요: MCP는 연결성을 위해, 스킬은 절차적 지식을 위해.

MCP에 대한 자세한 내용을 알아보고 MCP 서버 구축 방법에 대한 문서를 확인하세요.

## 어떻게 함께 작동하는가

진정한 힘은 이러한 구성 요소들을 결합할 때 나타납니다. 각각은 고유한 목적을 수행하며, 함께 정교한 에이전트 워크플로우를 만듭니다.

### **비교: 올바른 도구 선택**

| 기능 | 스킬 | 프롬프트 | 프로젝트 | 서브에이전트 | MCP |
| --- | --- | --- | --- | --- | --- |
| **제공 내용** | 절차적 지식 | 순간별 지침 | 배경 지식 | 작업 위임 | 도구 연결성 |
| **지속성** | 대화 간 | 단일 대화 | 프로젝트 내 | 세션 간 | 지속적 연결 |
| **포함 내용** | 지침 + 코드 + 자산 | 자연어 | 문서 + 컨텍스트 | 전체 에이전트 로직 | 도구 정의 |
| **로드 시점** | 동적, 필요 시 | 매 턴 | 프로젝트 내 항상 | 호출 시 | 항상 사용 가능 |
| **코드 포함 가능 | 예 | 아니오 | 아니오 | 예 | 예 |
| **가장 적합한 용도** | 전문 지식 | 빠른 요청 | 중앙화된 컨텍스트 | 전문화된 작업 | 데이터 액세스 |

### **예시 에이전트 워크플로우: 리서치 에이전트**

여러 구성 요소를 결합하는 포괄적인 리서치 에이전트를 구축해 봅시다. 이 예시는 경쟁 분석을 위해 에이전트를 조립하고 활성화하는 방법을 보여줍니다.

프로젝트 지침 추가:

## 검색 모범 사례

**5단계: 리서치 에이전트 활성화**

이제 Claude에게 다음과 같이 물어볼 때: "우리 상위 3개 경쟁사가 새로운 AI 기능을 어떻게 포지셔닝하는지 분석하고 우리가 활용할 수 있는 격차를 식별하세요"

다음이 발생합니다:

1. **프로젝트 컨텍스트 로드:** Claude가 업로드된 리서치 문서에 접근하고 프로젝트 지침을 따름
2. **MCP 연결 활성화:** Claude가 최신 경쟁사 브리핑을 위해 Google Drive를 검색하고 GitHub 데이터를 가져옴
3. **스킬 참여:** 경쟁 분석 스킬이 분석 프레임워크 제공
4. **서브에이전트 실행** (Claude Code에서): market-researcher가 산업 데이터를 수집하는 동안 technical-analyst는 기술 구현을 검토
5. **프롬프트 수정:** 대화적 지침 제공: "특히 헬스케어 분야의 기업 고객에 집중하세요"

**결과:** 여러 데이터 소스에서 인용하고, 분석 프레임워크를 따르고, 전문 지식을 활용하며, 리서치 프로젝트 전반에 걸쳐 컨텍스트를 유지하는 포괄적인 경쟁 분석.

## 자주 묻는 질문

### **스킬은 어떻게 작동하나요?**

스킬은 점진적 공개를 사용하여 Claude를 효율적으로 유지합니다. 작업 처리 시, Claude는 관련 일치 항목을 식별하기 위해 먼저 스킬 메타데이터(설명 및 요약)를 스캔합니다. 스킬이 일치하면 Claude는 전체 지침을 로드합니다. 마지막으로, 스킬이 실행 가능한 코드나 참조 파일을 포함하는 경우, 필요할 때만 로드됩니다.

이 아키텍처는 많은 스킬을 사용할 수 있으면서도 Claude의 컨텍스트 창을 압도하지 않는다는 것을 의미합니다. Claude는 필요할 때 정확히 필요한 것에 접근합니다.

### **스킬 vs 서브에이전트: 언제 무엇을 사용해야 하나요**

**스킬을 사용할 때:** 모든 Claude 인스턴스가 로드하고 사용할 수 있는 기능을 원할 때. 스킬은 교육 자료와 같습니다 - 이들은 모든 대화에서 특정 작업에 Claude를 더 좋게 만듭니다.

**서브에이전트를 사용할 때:** 특정 목적을 위해 설계된 독립 작업 워크플로우를 처리하는 완전하고 자립적인 에이전트가 필요할 때. 서브에이전트는 자체 컨텍스트와 도구 권한을 가진 전문 직원과 같습니다.

**함께 사용할 때:** 전문 지식이 있는 서브에이전트를 원할 때. 예를 들어, 코드 검토 서브에이전트는 특정 언어 모범 사례에 대한 스킬을 사용하여 서브에이전트의 독립성과 스킬의 이식 가능한 전문 지식을 결합할 수 있습니다.

### **스킬 vs 프롬프트: 언제 무엇을 사용해야 하나요**

**프롬프트를 사용할 때:** 일회성 지침을 제공하거나, 즉각적 컨텍스트를 제공하거나, 대화적 왕복을 할 때. 프롬프트는 반응적이고 일시적입니다.

**스킬을 사용할 때:** 반복적으로 필요한 절차나 전문 지식이 있을 때. 스킬은 선제적입니다 - Claude는 언제 적용할지 알고 - 대화 간에 지속됩니다.

**함께 사용:** 프롬프트와 스킬은 자연스럽게 상호 보완합니다. 스킬을 사용하여 기초 전문 지식을 제공한 다음, 각 작업을 위해 특정 컨텍스트와 수정을 제공하기 위해 프롬프트를 사용하세요.

### **스킬 vs 프로젝트: 언제 무엇을 사용해야 하나요**

**프로젝트를 사용할 때:** 특정 이니셔티브에 대한 모든 대화에 영향을 미쳐야 하는 배경 지식과 컨텍스트가 필요할 때. 프로젝트는 항상 로드되는 정적 참조 자료를 제공합니다.

**스킬을 사용할 때:** 관련이 있을 때만 활성화되는 절차적 지식과 실행 가능한 코드가 필요할 때. 스킬은 요청 시 로드되는 동적 전문 지식을 제공하여 컨텍스트 창을 절약합니다.

**함께 사용할 때:** 지속적 컨텍스트와 전문 기능을 모두 원할 때. 예를 들어, 제품 사양과 사용자 리서치를 포함하는 "제품 개발" 프로젝트를 기술 문서 생성과 사용자 피드백 데이터 분석을 위한 스킬과 결합하세요.

**주요 차이점:** 프로젝트는 "알아야 할 것"을 말합니다. 스킬은 "어떻게 하는지"를 말합니다. 프로젝트는 작업할 지식 기반을 제공합니다. 스킬은 어디서든 작동하는 기능을 제공합니다 - 모든 대화, 모든 프로젝트.

### **서브에이전트가 스킬을 사용할 수 있나요?**

예. Claude Code와 Agent SDK에서, 서브에이전트는 메인 에이전트처럼 스킬에 접근하고 사용할 수 있습니다. 이는 전문화된 서브에이전트가 이식 가능한 전문 지식을 활용하는 강력한 조합을 만듭니다.

예를 들어, python-developer 서브에이전트는 팀 규칙에 따라 데이터 변환을 수행하기 위해 pandas-analysis 스킬을 사용할 수 있고, documentation-writer 서브에이전트는 일관된 API 문서 형식화를 위해 technical-writing 스킬을 사용할 수 있습니다.
