## Système d'amélioration de contenu basé sur l'IA

### Extraction de concepts : Compréhension au-delà des mots-clés

Contrairement au simple appariement de mots-clés, la compétence utilise Claude IA pour extraire la **compréhension sémantique** du contenu de l'article :

**Comment ça marche** :
```typescript
// L'IA analyse le contenu, pas juste les motifs
const concepts = await askClaude(`
  Extraire 5-10 concepts techniques clés de cet article.
  Pour chaque concept fournir :
  - term : Nom exact du concept
  - definition : Brève explication
  - isMainTopic : Est-ce le sujet principal ?
  - importance : Score 1-10
`, articleContent);
```

**Impact réel** :
- ✅ **Distingue** : "Skills" (fonctionnalité Claude) vs. "skills" (capacités)
- ✅ **Comprend le contexte** : "Python" en codage vs. "python" en zoologie
- ✅ **Évalue l'importance** : Sujet principal (10/10) vs. mentionné (3/10)
- ✅ **Permet l'intelligence** : Articles connexes, références croisées, parcours d'apprentissage

### Insertion intelligente de références croisées

La compétence ne se contente pas de lier ; elle **comprend où lier aide les lecteurs** :

**Processus de décision IA** :
1. Analyse la structure des paragraphes et le contexte
2. Détermine le niveau de connaissances actuel du lecteur
3. Insère les liens où elles sont les plus utiles (pas première occurrence mécanique)
4. Évite les liens excessifs (limite à 3-5 liens par article)
5. Ignore les endroits inappropriés (titres, blocs de code, citations)

**Exemple : Approche IA vs Naïve** :
```
Entrée : "La fonctionnalité Skills de Claude aide les développeurs à construire des agents puissants"

Approche naïve : Lier la première occurrence de "Skills"
Approche IA : Comprendre que "Claude" et "agents" sont les concepts nécessitant explication
Résultat : Lier "Skills" dans le contexte, pas juste première apparition
```

### Recommandations intelligentes d'articles connexes

Au-delà du simple appariement de tags, l'IA crée des **parcours de lecture intelligents** :

**Critères d'analyse** :
- **Chevauchement de concepts** : Concepts techniques partagés
- **Progression des connaissances** : Débutant → Intermédiaire → Avancé
- **Relations de prérequis** : L'article A permet la compréhension de l'article B
- **Connaissances complémentaires** : Article A + Article B = image complète

## Système de traitement intelligent d'images

### Le défi des images : Signal vs. Bruit

La plupart des articles contiennent 15-20 images, mais seulement 1-3 fournissent une valeur réelle. La compétence implémente le **filtrage heuristique** qui atteint 80-90% de réduction du bruit :

**Algorithme de filtrage** :
```typescript
// Liste noire : Rejet immédiat
const blacklist = [
  'placeholder.svg', 'favicon', 'logo', 'spinner',
  'avatar', 'decoration', 'icon-', 'social-share'
];

// Liste blanche : Doit conserver
const whitelist = [
  'diagram', 'chart', 'screenshot', 'visualization',
  'architecture', 'flowchart', 'graph', 'timeline'
];

// Règles contextuelles
if (isNearContext(img, ['diagram', 'figure', 'example'])) {
  return true; // Conserver les images de contenu
}
```

### Trois stratégies de traitement d'images

Basées sur les capacités de la source, la compétence choisit automatiquement la stratégie optimale :

**Stratégie A : URLs externes (support CORS)**
- Utilisée quand la source supporte les requêtes cross-origin
- Aucun téléchargement requis, référence les URLs originales
- Testée et vérifiée pour Claude.com, GitHub, plateformes majeures

**Stratégie B : Téléchargement local (compatibilité universelle)**
- Télécharge les images vers `public/images/docs/{slug}/`
- Fonctionne 100% du temps, fournit disponibilité hors ligne
- Utilise l'espace de stockage local mais élimine les dépendances externes

**Stratégie C : Détection automatique (choix intelligent)**
- Teste la première image pour le support CORS
- Sélectionne automatiquement la stratégie optimale
- Fournit le meilleur des deux mondes avec zéro configuration

**Exemple réel : Traitement Claude.com** :
```bash
# Test du support CORS
curl -I "https://cdn.example.com/mcp-diagram.png"
# Réponse : access-control-allow-origin: *
# Décision : Utiliser les URLs externes (pas de téléchargement nécessaire)
```

## Programmation défensive : Traitement sécurisé du contenu

### Le problème : Sources inconnues, risques inconnus

Les articles proviennent de sources diverses—Anthropic, GitHub, Medium, blogs personnels—chacune avec différentes bibliothèques de composants et saveurs markdown. L'analyse traditionnelle planterait sur une syntaxe inconnue.

### La solution : Pipeline de traitement sécurisé du contenu

La compétence implémente un système de **traitement sécurisé en trois phases** :

**Phase 1 : Gestion des composants inconnus**
```typescript
// Liste blanche des composants Fumadocs connus
const fumadocsComponents = [
  'Callout', 'Cards', 'Card', 'Tabs', 'Tab',
  'Steps', 'Step', 'Files', 'Folder', 'File'
];

// Dégradation sécurisée des composants inconnus
content = content.replace(
  /&lt;([A-Z][a-zA-Z]*)[^&gt;]*&gt;([\s\S]*?)&lt;\/\1&gt;/g,
  (match, componentName, innerContent) => {
    if (fumadocsComponents.includes(componentName)) {
      return match; // Conserver les composants connus
    }
    return `&lt;!-- Original : &lt;${componentName}&gt; --&gt;\n${innerContent}`;
  }
);
```

**Phase 2 : Corrections des pièges MDX**
- Corrige les motifs `<5k` → `&lt;5k`
- Gère les balises HTML dangereuses dans le texte
- Corrige les problèmes d'espacement du formatage gras
- Détecte les balises JSX non appariées

**Phase 3 : Injection automatique d'imports**
- Détecte les composants Fumadocs utilisés
- Injecte automatiquement les imports nécessaires
- Prévient les erreurs "Component is not defined"
- Maintient un code propre et maintenable

## Dépannage et optimisation

### Problèmes courants et solutions

**Problème** : "Les images ne se traitent pas correctement"
- **Cause** : Paramètre `withAllImages: true` manquant dans l'appel Jina MCP
- **Solution** : La compétence inclut une validation obligatoire qui prévient cette erreur
- **Prévention** : Toujours utiliser la validation intégrée de la compétence

**Problème** : "La qualité de traduction varie"
- **Cause** : Non-préservation des termes techniques
- **Solution** : La compétence inclut une liste complète de préservation des termes
- **Optimisation** : Personnaliser les termes préservés pour votre domaine

**Problème** : "Échec de la construction après importation"
- **Cause** : Erreurs de syntaxe MDX de composants inconnus
- **Solution** : Le traitement défensif de la compétence prévient 90% des erreurs
- **Récupération** : Vérifier les avertissements de traitement et ajuster manuellement

### Optimisation des performances

**Pour les opérations à haut volume** :
```typescript
// Optimiser l'utilisation des tokens
const config = {
```

**Gestion de la mémoire** :
